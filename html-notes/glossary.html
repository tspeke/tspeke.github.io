    HTML Basics

  • HTML provides the backbone for a web page
  • <elementName> : Elements are the building blocks of a HTML document. Can represent headings, paragraphs, links, images...
  • Some of the most common Elements:
    • <h1> to <h6> : Headings
    • <p> : Paragraph
    • <img> : Image
    • <body> : Contains displayed content for HTML document
    • <section> : Divide content into sections
    • <div> : Generic container for holding other HTML elements
    • <a> : Anchor for URL links to web pages
    • <ul> and <ol> : Used with <li> nested for unordered and ordered lists
    • <em> : Emphasis
    • <strong> : Strong emphasis
    • <figure> : Groups content including <figcaption>
    • <main> : Main content for web page
    • <footer> : At end of HTML document
    • <script> : Embeds executable code
  • id : "Identifiers" - Should be used only once per HTML document
  • class : "Classes" - Group elements for styling and behaviour
  • HTML Boilerplate : Includes the basic structure and essential elements every HTML document needs

    • Audio and Videos

  • <audio> : Inserts sound content into HTML document. Supports .mp3, .wav and .ogg
    • src : Directs to location of audio file
    • controls : Adds ability to play/pause and mute audio
    • Note that some browsers support different file types and may have different default settings
    • loop : Automatically restarts audio once finished
    • muted : Boolean attribute which sets audio default to be muted
  • <source> : Source elements can be nested within media element. This causes the browser to select the first source that it understands
  • <video> : Inserts video content into HTML document. Supports .mp4, .webm and .ogg
    • All attributes introduced in <audio> element are supported in <video>
    • autoplay : Boolean attribute which causes video to play on loading
    • width : Sets the width of the video screen
    • poster : Displays an image with the destination given when video is downloading

    • Target, Links and Paths

  • target : Attribute on anchor and link elements. Tells the browser where to open the URL for the element.
    • There are four possible values for the attribute:
    • _self : Default value. Opens link in the current "browsing context" (i.e. current tab and window)
    • _blank : Opens link in a new "browsing context" --> Typically a new tab but for some browser configurations will be a new window
    • _parent : Opens link in parent of current context. E.g. if opening a link from an iframe --> _parent would open the link in the tab not in the embedded frame
    • _top : Opens link in the top-most "browsing context". E.g. will open in full browser tab/window even if nested in embedded frames
  • "Absolute Path" : Complete link to a resource relative to the root of the directory. Also includes the "Protocol" i.e. http, https or file for locally stored data - these are all followed by __://. As well as the file and domain names
  • "Relative Path" : Specifies the location of a file relative to the directory of the current file
  • General Rules for which to Use:
    • Absolute when linking to resources hosted on external website
    • Absolute when want link to a page or resource to work consistently regardless of document location within site
    • Relative when linking to resources within same website
    • Relative to keep code cleaner during development
    • Relative during local testing to ensure links work without internet connection
  • Path Syntax:
    • / or \ (dependant on OS) : "Path Separator". Indicates subsequent files are held within previous folder locations
    • . : Single dot points to --> Current directory
    • .. : Double dot points to --> Parent directory
  • <a> : "Anchor" element --> when clicked takes user to specified URL
    • href :
    • target :
    • :link : Default state representing a link the user has not yet interacted with
    • :visited : State which applies when the user has already visited the page being linked to
    • :hover : State which applies when the user is hovering their cursor over the link
    • :focus : State which applies when we focus on a link i.e. using the tab key
    • :active : State which applies when the link is being activated by the user. Usually corresponding to left clicking on link
    • When using these states to style links there is a specific order you must write your CSS in: :link, :visited, :hover, :focus then :active

    • HTML Forms

  • <form> : Creates an HTML form for user input
    • action : Specifies URL where form data should be sent once submitted
    • method : Specifies HTTP method to use when sending form data. Most common methods being GET and POST
  • <input> : Void element that creates an input field for user input
    • type : Specifies type of input field. E.g:
      • text :
      • email :
      • number :
      • radio :
      • checkbox :
      • textarea :
      • button :
    • placeholder : Displays text hinting at what user should type into input field
    • value : Used to specify the value of the input
    • name : Assigns a name to the input field serving as the heading key when the data is submitted. For radio buttons, assigning the same name allows only one to be selected at once
    • size : Defines the number of characters that should be visable as user types into input (i.e. the size of the input box)
    • min : Can be used with input types such as number to specify the minimum value allowed in the input field
    • max : Can be used with input types such as number to specify the maximum value allowed in the input field
    • minlength : Specifies the minimum number of characters required in input field
    • maxlength : Specifies the maximum number of characters allowed in input field
    • required : Boolean attribute specifying that input field must be filled before form can be submitted
    • disabled : Boolean attribute specifying that input field should be disabled
    • readonly : Boolean attribute specifying input field is read-only
  • <label> : Creates a label for an input field
    • for : Specifies which input field the label is for
    • Implicit form association is where inputs can be associated with labels by wrapping the input field inside the label element
    • Explicit form association is where inputs are associated to corresponding labels using the for attribute on the label element and id attribute on the input element
  • <button> : Creates a clickable button
    • type : Controls the behaviour of the button when activated. E.g:
      • submit : Submits the form
      • reset : Resets all fields in the form
      • button : Can program alternate functions by linking to Java Script
  • <fieldset> : Groups related inputs together
  • <legend> : Adds a caption to describe the group of inputs
  • Focused State : State of the input field when it is currently selected by the user

  • <tfoot> : Groups the footer content in an HTML table
  • <tr> : Creates a row
  • <th> : Creates a header cell
  • <td> : Creates a data cell
  • <caption> : Adds a title
