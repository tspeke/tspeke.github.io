HTML Boilerplate

The link element

The link element is used to link to external sources such as stylesheets and site icons.For example here is the syntax for a link to an external CSS file:

<link rel="stylesheet" href="./styles.css" />

The meaning of each part are as follows:

  • rel : Attribute specifying relationship between linked document and HTML document
    • In this case specified linked document is of "spreadsheet" type
    • It is considered best practice to keep your HTML and CSS files separate then link the CSS file to the HTML document
  • href : Attribute specifying the location of the URL for the external resource
  • ./ : The dot followed by a forward slash in the example tells the computer to look in the current folder for the styles.css file
  • The link element should be put inside the head element

Often link to external fonts, stylsheets and icons. Google Fonts is a set of free and open source custom fonts. You can choose which fonts you'd like to use and then Google will provide the necessary HTML and CSS code:

<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link href="https://fonts.googleapis.com/css2?family=Playwrite+CU:wght@100..400&display=swap" rel="stylesheet" />

Here "preconnect" : Value for the rel attribute tells browser to make an early connection to the link in the href attribute

Another common use of the element is linking to a favicon:

<link rel="icon" href="favicon.ico" />

A favicon is a small icon typically displayed next to the browser tab next to the page name

Introduction of HTML Boilerplate

A boilerplate is a ready-made template for a webpage. A boilerplate contains the basic structure and essential elements every HTML document needs. It saves time and ensures pages are set up properly.

Here is an example boilerplate:

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="utf-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title>freeCodeCamp</title>
        <link rel="stylesheet" href="./styles.css" />
    </head>
    <body>
    </body>
</html>

Breaking it down...

  • DOCTYPE declaration : Tells browser which version of HTML you are using
  • html tag : Wraps around all content specifying the coding language of the contents
  • head section : Contains behind-the-scenes information including metadata and page formatting details. head contains:
    • meta element : Has details and is how other websites preview the page's link
    • title element : Specifies the text which will appear in the tab
    • link element : Link external spreadsheets and icons
  • body section : Where all the page content goes

The boilerplate ensures pages are structured correctly, work accross multiple platforms (browsers) and avoid common mistakes by following best practices. Can customise the boilerplate to meet specific needs of a page.
